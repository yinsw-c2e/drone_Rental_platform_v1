## 项目概述

无人机租赁平台的派单任务表(DispatchTask)是整个智能匹配与派单系统的核心数据结构。该系统实现了从订单创建到任务执行的完整闭环管理，包括任务池匹配、正式派单、飞手执行、飞行监控等功能模块。

系统采用三层架构设计：前端移动应用负责用户交互，后端服务提供业务逻辑，数据库存储所有核心数据。派单任务表作为核心实体，承载着订单关联、飞手分配、状态管理、调度算法等关键功能。

## 核心组件

### 派单任务表结构

派单任务表(DispatchTask)是系统的核心实体，包含以下关键字段：

**基础标识字段**
- `id`: 主键自增ID
- `task_no`: 任务编号，唯一索引
- `order_id`: 关联订单ID
- `client_id`: 业主ID

**任务类型与优先级**
- `task_type`: 任务类型(instant, scheduled, batch)
- `priority`: 优先级1-10，默认5

**状态管理**
- `status`: 任务状态(pending, matching, dispatching, assigned, cancelled, expired)

**货物信息**
- `cargo_weight`: 货物重量(kg)
- `cargo_volume`: 货物体积(立方厘米)
- `cargo_category`: 货物类别
- `is_hazardous`: 是否危险品

**位置信息**
- `pickup_*`: 取货点经纬度和地址
- `delivery_*`: 送货点经纬度和地址
- `flight_distance`: 飞行距离(km)

**时间约束**
- `required_*`: 要求取货/送达时间
- `time_window_*`: 时间窗口
- `dispatch_deadline`: 派单截止时间

**预算约束**
- `budget_min/max`: 预算范围(分)
- `offered_price`: 业主出价(分)

**匹配要求**
- `required_license_type`: 执照类型要求
- `min_pilot_rating`: 飞手最低评分
- `min_drone_rating`: 无人机最低评分
- `min_credit_score`: 最低信用分

**派单结果**
- `assigned_pilot_id`: 分配飞手ID
- `assigned_drone_id`: 分配无人机ID
- `assigned_owner_id`: 分配机主ID
- `assigned_at`: 分配时间
- `final_price`: 最终成交价(分)
- `match_score`: 匹配得分
- `match_details`: 匹配详情(JSON)

**匹配统计**
- `match_attempts`: 匹配尝试次数
- `max_attempts`: 最大尝试次数
- `last_match_time`: 最后匹配时间
- `fail_reason`: 失败原因

### 派单任务状态管理

派单任务采用有限状态机设计，包含以下状态转换：

```mermaid
stateDiagram-v2
[*] --> 待分配 : 创建任务
待分配 --> 匹配中 : 触发智能匹配
匹配中 --> 已分配 : 飞手接受任务
匹配中 --> 待分配 : 匹配失败
已分配 --> 执行中 : 飞手开始执行
执行中 --> 已完成 : 任务完成
执行中 --> 已取消 : 任务取消
待分配 --> 已取消 : 任务取消
已分配 --> 已取消 : 任务取消
匹配中 --> 已过期 : 超过截止时间
待分配 --> 已过期 : 超过截止时间
```

**状态业务含义**:
- `pending`: 待分配 - 任务刚创建，等待匹配
- `matching`: 匹配中 - 系统正在进行智能匹配
- `dispatching`: 派单中 - 已找到候选人，正在通知飞手
- `assigned`: 已分配 - 飞手接受任务，准备执行
- `cancelled`: 已取消 - 任务被取消
- `expired`: 已过期 - 超过截止时间未完成

### 智能匹配算法

系统采用多维度评分算法，计算飞手-无人机组合的匹配度：

```mermaid
flowchart TD
Start([开始匹配]) --> LoadTask[加载任务信息]
LoadTask --> SearchPilots[搜索可用飞手]
SearchPilots --> CalcDistance[计算距离得分]
CalcDistance --> CalcLoad[计算载荷匹配]
CalcLoad --> CalcQualification[计算资质匹配]
CalcQualification --> CalcCredit[计算信用得分]
CalcCredit --> CalcPrice[计算价格匹配]
CalcPrice --> CalcTime[计算时间匹配]
CalcTime --> CalcRating[计算服务评分]
CalcRating --> SumScore[计算总分]
SumScore --> FilterScore{过滤最低分数}
FilterScore --> |通过| AddCandidate[添加候选人]
FilterScore --> |不通过| NextPilot[下一个飞手]
AddCandidate --> NextPilot
NextPilot --> MorePilots{还有飞手?}
MorePilots --> |是| SearchPilots
MorePilots --> |否| SortCandidates[排序候选人]
SortCandidates --> LimitCandidates[限制候选数量]
LimitCandidates --> End([完成匹配])
```

**匹配算法权重**:
- 距离得分: 25分权重
- 载荷匹配: 15分权重  
- 资质匹配: 20分权重
- 信用得分: 15分权重
- 价格匹配: 10分权重
- 时间匹配: 10分权重
- 服务评分: 5分权重
