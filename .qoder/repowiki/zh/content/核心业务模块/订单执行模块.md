## 项目概述

订单执行模块是无人机租赁平台的核心业务模块，负责管理从需求确认到任务完成的完整订单生命周期。该模块实现了复杂的订单状态管理、履约执行跟踪、飞行监控以及多方协作机制。

### 执行阶段状态推进规则

| 状态变更 | 触发方 | 触发方式 |
|----------|--------|----------|
| `已指派执行 → 准备中` | 执行飞手 | 飞手在派单任务中点击"开始准备" |
| `准备中 → 飞行中` | 系统 | 飞手上报起飞或系统检测到首个飞行位置点 |
| `飞行中 → 已投送` | 执行飞手 | 飞手到达目的地后点击"确认投送" |
| `已投送 → 已完成` | 客户/系统 | 客户确认签收；或投送后24小时无操作，系统自动确认 |

## 事务处理机制

### 事务边界设计

订单系统采用严格的事务边界设计，确保数据一致性：

```mermaid
flowchart TD
A[订单创建] --> B{事务开始}
B --> C[验证无人机状态]
C --> D[检查幂等性]
D --> E[计算费用]
E --> F[创建订单记录]
F --> G[记录时间线]
G --> H{提交事务}
H --> I[订单创建成功]
J[订单状态变更] --> K{事务开始}
K --> L[验证权限]
L --> M[更新订单状态]
M --> N[更新相关表]
N --> O[记录时间线]
O --> P{提交事务}
P --> Q[状态变更成功]
```

### 事务隔离级别

系统采用以下事务隔离策略：

1. **订单创建事务**：包含无人机验证、幂等性检查、费用计算、订单创建等所有操作
2. **状态变更事务**：包含权限验证、状态更新、相关表更新、时间线记录
3. **退款事务**：包含退款记录创建、支付状态更新、余额调整

### 退款计算逻辑

退款金额基于以下规则计算：

| 取消时间 | 退款比例 | 说明 |
|----------|----------|------|
| 提前24小时以上 | 100% | 全额退款，包括订单金额和押金 |
| 0-24小时前 | 70% | 退款70%订单金额 + 全额押金 |
| 已过开始时间 | 0% | 无法取消，无退款 |

## 争议解决机制

### 争议类型分类

系统支持多种争议类型：

| 争议类型 | 适用场景 | 处理流程 |
|----------|----------|----------|
| 货物损坏 | 货物在运输过程中损坏 | 客户发起 → 平台介入 → 责任认定 → 退款/赔偿 |
| 飞行事故 | 无人机在执行过程中发生事故 | 飞手报告 → 平台调查 → 责任划分 → 保险理赔 |
| 服务质量 | 服务未达到约定标准 | 客户投诉 → 平台调查 → 责任认定 → 补偿措施 |
| 支付纠纷 | 支付相关争议 | 用户申诉 → 平台核实 → 责任判定 → 处理决定 |

## 故障排除指南

### 常见问题诊断

#### 订单状态异常
**症状**：订单状态显示异常或无法更新
**排查步骤**：
1. 检查订单状态变更日志
2. 验证权限验证逻辑
3. 确认事务提交状态
4. 检查并发控制机制

#### 支付处理失败
**症状**：支付状态与预期不符
**排查步骤**：
1. 检查支付回调处理
2. 验证第三方支付接口
3. 确认退款流程执行
4. 检查支付状态同步

#### 飞行数据缺失
**症状**：飞行监控数据不完整
**排查步骤**：
1. 检查飞行数据上报频率
2. 验证地理位置服务
3. 确认网络连接状态
4. 检查数据存储配置
